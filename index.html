HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • The body element is an example of a semantic HTML element.
  • In semantic HTML elements, the name of the element describes its content and the role it plays on a webpage making it easier to read.
  • The h1 tag should only appear once on each page.
  • Alt text is important for screen readers and SEO.

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • Linking to an external stylesheet in the head section of your html is preferred to inline styling or styling in the head section.
  • Separation of concerns is a widely used term in web development and refers to separating code into different files based on use.
  • Do Not Repeat Yourself (DRY) is a coding principle that encourages developers to reuse or share pieces of code (ie, grouping tags in css).

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git add -A: adds all files in local branch into staging for the next commit
  • git commit -m "text": commit staged files to a commit snapshot with attached message
  • git pull origin main: pulls files from remote branch to local branch (good practice before pushing commits)
  • git push origin working/branch: sends local branch commits to remote branch in repository

JavaScript

  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
  • It's important to remember control flow so that you aren't trying to access code before it's being read by the computer.
